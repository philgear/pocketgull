# Data & Privacy

Understanding how clinical information flows through Pocket Gull is critical for building practitioner trust. The platform operates as a **localized clinical data processor** with no centralized remote database.

## Storage Model

Pocket Gull follows a **Privacy by Design** architecture:

- **Local Persistence** — All patient states, clinical brackets, and historical visit notes are stored strictly within the client's local browser session.
- **No Remote Database for Records** — There is no centralized server or cloud database storing persistent patient clinical records.
- **Transient AI Processing** — Clinical context transmitted to <DocNode term="Vertex AI" category="Google Cloud" hint="Enterprise-grade AI platform. Ensures data residency and HIPAA compliance." link="https://cloud.google.com/vertex-ai" linkLabel="Vertex API →" icon="https://www.gstatic.com/devrel-devsite/prod/v0e0f589edd85502a40d78d7d0825db8ea5ef3b99ab4070381571c97ab7df9861/cloud/images/favicons/onecloud/favicon.ico">Vertex AI</DocNode> and <DocNode term="@google/adk" category="Google · Agent Development Kit" hint="Multi-agent orchestration framework. Agents process clinical context in-memory; no remote persistence." link="https://google.github.io/adk-docs/" linkLabel="ADK Docs →" icon="https://ai.google.dev/favicon.ico">ADK</DocNode> agents is used solely for immediate generation. Data is not retained by the application backend or used for model training.
- **Immutable Auditing** — While records remain local, clinical interactions are securely logged to a centralized <DocNode term="BigQuery" category="Google Cloud" hint="Serverless, highly scalable enterprise data warehouse." link="https://cloud.google.com/bigquery/docs" linkLabel="BigQuery Docs →" icon="https://www.gstatic.com/devrel-devsite/prod/v0e0f589edd85502a40d78d7d0825db8ea5ef3b99ab4070381571c97ab7df9861/cloud/images/favicons/onecloud/favicon.ico">BigQuery</DocNode> dataset to establish an auditable chain of custody.
- **Real-time Indicators** — Visual "Saving…" / "Saved ✔" feedback confirms local persistence status.

## HIPAA Compliance Readiness

Pocket Gull's enterprise deployment architecture is designed to support HIPAA compliance:

1. **Business Associate Agreement (BAA)** — Deployment on Google Cloud Run, Vertex AI, and BigQuery is covered under standard GCP BAAs.
2. **Data Residency** — Vertex AI endpoints ensure inference data remains within designated geographic boundaries (e.g., `us-west1`).
3. **PHI Minimization** — All requests sent to non-HIPAA compliant external services (like Google CSE) are aggressively minimized, stripping names, birthdates, and explicit identifiers before transmission.
4. **Audit Logger** — Every AI invocation and data modification triggers an immutable telemetry event via the `AuditService` directly into BigQuery, capturing user IDs and timestamped actions for compliance review.

## Data Export & Portability

### FHIR Bundles
Patient data is exportable as standard <DocNode term="FHIR R4" category="HL7 International" hint="Fast Healthcare Interoperability Resources Release 4 — the current HL7 standard for electronic health data exchange." link="https://hl7.org/fhir/R4/" linkLabel="HL7 FHIR R4 →" icon="https://hl7.org/favicon.ico">**FHIR R4 JSON Bundles**</DocNode>, Unicode-safe and Base64-encoded. This ensures open data portability across any FHIR-compliant healthcare system.

### FHIR Import
The Import Service accepts FHIR payloads — including **Epic/MyChart "Lucy" format** — mapping external `DiagnosticReport`, `Observation`, and `Condition` resources into Pocket Gull's native models.

### Printable Stationery
Generated insights can be physically printed via CSS-optimized layouts featuring Halftone diagnostic maps. Sensitive records can be kept **strictly on offline paper** when required.
