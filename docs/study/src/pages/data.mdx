# Data & Privacy

Understanding how clinical information flows through Pocket Gull is critical for building practitioner trust. The platform operates as a **localized clinical data processor** with no centralized remote database.

## External Research Sources

To augment clinical reasoning **without compromising patient privacy**, Pocket Gull integrates with authoritative medical data sources:

### NCBI PubMed E-utilities
Queries peer-reviewed medical literature via the <DocNode term="PubMed E-utilities" category="NIH · National Library of Medicine" hint="NCBI's programmatic interface to PubMed. Powers the SSR backend and API proxy." link="https://expressjs.com/" linkLabel="expressjs.com →" icon="https://expressjs.com/images/favicon.png">Express.js</DocNode> `/api/pubmed` endpoint.

### Google Programmable Search Engine
Surfaces relevant differential diagnostic information via <DocNode term="Google CSE" category="Google" hint="Customizable search engine for specific medical domains." link="https://programmablesearchengine.google.com/" linkLabel="CSE Console →" icon="https://www.google.com/favicon.ico">Google CSE</DocNode>. Rendered in a sandboxed Research Frame.

> **PHI Protection**: All queries sent to external sources are anonymized and stripped of protected health information prior to transmission.

## Storage Model

Pocket Gull follows a **Privacy by Design** architecture:

- **Local Persistence** — All patient states, clinical brackets, and historical visit notes are stored strictly within the client's local browser session.
- **No Remote Database** — There is no centralized server or cloud database storing persistent patient records.
- **Transient AI Processing** — Clinical context transmitted to <DocNode term="Gemini 2.5 Flash" category="Google DeepMind" hint="Fast, cost-efficient multimodal model. Data is used solely for immediate inference and is not retained." link="https://ai.google.dev/gemini-api/docs/models#gemini-2.5-flash" linkLabel="Model Reference →" icon="https://ai.google.dev/favicon.ico">Gemini</DocNode> and <DocNode term="@google/adk" category="Google · Agent Development Kit" hint="Multi-agent orchestration framework. Agents process clinical context in-memory; no remote persistence." link="https://google.github.io/adk-docs/" linkLabel="ADK Docs →" icon="https://ai.google.dev/favicon.ico">ADK</DocNode> agents is used solely for immediate generation. Data is not retained by the application backend.
- **Real-time Indicators** — Visual "Saving…" / "Saved ✔" feedback confirms local persistence status.

## Data Export & Portability

### FHIR Bundles
Patient data is exportable as standard <DocNode term="FHIR R4" category="HL7 International" hint="Fast Healthcare Interoperability Resources Release 4 — the current HL7 standard for electronic health data exchange." link="https://hl7.org/fhir/R4/" linkLabel="HL7 FHIR R4 →" icon="https://hl7.org/favicon.ico">**FHIR R4 JSON Bundles**</DocNode>, Unicode-safe and Base64-encoded. This ensures open data portability across any FHIR-compliant healthcare system.

### FHIR Import
The Import Service accepts FHIR payloads — including **Epic/MyChart "Lucy" format** — mapping external `DiagnosticReport`, `Observation`, and `Condition` resources into Pocket Gull's native models.

### Printable Stationery
Generated insights can be physically printed via CSS-optimized layouts featuring Halftone diagnostic maps. Sensitive records can be kept **strictly on offline paper** when required.
