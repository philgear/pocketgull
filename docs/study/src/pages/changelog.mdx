## v1.0.0-rc6 — 2026-06-14

**CodeQL Security Hardening (Full Remediation) & Rate Limiting**

### Fixed
- **[CodeQL] SSRF Critical × 5 (dicom.ts)**: Added `sanitiseDicomParam()` (allowlist: `[a-zA-Z0-9._-]`) and `validateDicomUid()` (digits and dots only, per DICOM standard) helpers. All five DICOM route handlers (`/studies`, `/rendered`, `/store`, `/raw`, `/delete`) now sanitise every user-provided query param before URL construction — no raw `req.query` value ever reaches `fetch()`.
- **[CodeQL] SSRF (server.ts)**: Sanitised `projectId` + `location` Vertex AI URL construction with char-allowlist regex; `rawModel` is strictly allowlisted via `normalizeAndValidateModel()`.
- **[CodeQL] Missing Rate Limiting × 4 (server.ts)**: Replaced custom in-memory `rateLimiter()` with `express-rate-limit`'s `rateLimit()` on all flagged file-system-accessing routes.
- **[CodeQL] Path Traversal × 2 (server.ts `/docs/study`)**: Removed custom `resolve()`-based path handler entirely. `/docs/study` is now served exclusively via `express.static(studyDocsRoot)` — no user-controlled value reaches any file path computation.
- **[CodeQL] Clear-Text Logging (phi_compliance_scanner.py)**: Discarded the tainted regex match tuple element with `_` and removed it from every `print()` format string. Only violation type, file path, line number, and description are logged — no sensitive match data reaches any log sink.
- **[Pre-emptive] DICOM Router Rate Limiting**: Applied `express-rate-limit` at the `dicomRouter` level (60 req/min per IP) covering all DICOM proxy routes.
- **[Pre-emptive] Healthcare Router Rate Limiting**: Applied `express-rate-limit` at the `healthcareRouter` level (30 req/min per IP) covering all FHIR and Healthcare API proxy routes.
