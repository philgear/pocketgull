### Care Plan Recommendation Engine

A professional clinical analysis engine that synthesizes structured strategies for patient care. Output is organized by diagnostic lens, and each report node supports **inline agent queries** — click any recommendation to ask the AI directly about it.

### Orthomolecular Profiling & Biomarker Matrix

Automatically extracts and visualizes biochemical markers (e.g., Magnesium, B12, Homocysteine) from the AI-generated functional protocols. The glassmorphic dashboard uses pulsed indicators and semantic color-coding to provide an instant orthomolecular status read.

### PhysioNet 2026 Digital Signal & Electrophysiology Lens (`📡 PhysioNet Waveforms`)

High-frequency digital signal & electrophysiology lens ingesting EDF/PhysioNet waveform metrics: QRS interval duration (ms), neutral ST-segment elevation/depression (+0.04 mV), Fridericia QTc prolongation risk (418 ms), and Heart Rate Variability (HRV) spectral power density (LF/HF ratio 1.08, SDNN 58 ms).

### Colleague Collaboration Room (TaskFlow)

A real-time multiplayer workspace integrated directly into the patient's view. Clinicians can share patient states, dictate clinical notes, and chat collaboratively without leaving the active diagnostic session.

### Hands-Free Voice Dictation & Controls

Speak the wake word **"Gull"** followed by actions (e.g., *"Gull, highlight head"*, *"Gull, sync data"*) during active dictation. The application intelligently intercepts these commands without modifying the transcript, mapping them directly to UI actions. Clinicians can also use the integrated dictation modal to add tasks or chat messages completely hands-free.

### Good Samaritan Emergency Care & Relevancy Architecture

An offline emergency override mode engineered to maximize **Information Relevancy** when seconds count:
* **Cognitive Noise Suppression (95% UI Reduction):** Completely strips away non-essential UI (historical labs, herbal formulas, multi-tab billing) to present a single, high-contrast emergency card.
* **Imperative Resuscitation Directives:** Replaces complex medical jargon with 5-word physical action imperatives (*"PUSH HARD & FAST AT CENTER OF CHEST"*, *"TILT HEAD BACK, LIFT CHIN"*).
* **Visual & Haptic 110 BPM CPR Metronome:** Flashes high-visibility rhythmic cues matching AHA Basic Life Support (BLS) compression rate guidelines.
* **On-Device Zero-Latency AI (`window.ai` / Gemini Nano):** Operates 100% offline with full OpenTelemetry (OTel) network suppression, guaranteeing zero token latency during cellular blackouts.
* **EMT Hand-off QR Code Serialization (`lean-qr`):** Encodes emergency vitals, resuscitation timestamps, critical allergies, and anticoagulant warnings into an instant-scannable <DocNode term="FHIR R4" category="HL7 International" hint="Fast Healthcare Interoperability Resources — the current standard for electronic health data exchange." link="https://hl7.org/fhir/R4/" linkLabel="HL7 FHIR R4 →" icon="https://hl7.org/favicon.ico">FHIR R4</DocNode> Emergency Bundle.

### Client-Side Speech Barge-In

Local `onspeechstart` interruption tuning across all clinical dialog and voice assistant panels. When the clinician begins speaking or typing, audio playback is immediately muted and the response queue is cleared — eliminating the need to wait for the AI to finish speaking before asking a follow-up question.

### Cognition & Multilingual Care Plans

Seamlessly translate Care Plans into dyslexia-friendly or pediatric formats, or professionally translate them into **Spanish, German, French, Japanese, or Hindi** (aligned with global medical research exchange). Outputted to PDF using <DocNode term="jsPDF" category="James Hall · Parallax" hint="Client-side PDF generation library for JavaScript. Created by James Hall. ## Security & Compliance

### Vertex AI Enterprise Migration

The AI intelligence layer runs on regional <DocNode term="Vertex AI Enterprise" category="Google Cloud" hint="Google Cloud's enterprise-grade AI platform with IAM/ADC authentication, regional endpoints, custom safety thresholds, and SLA guarantees." link="https://cloud.google.com/vertex-ai" linkLabel="Vertex AI Docs →" icon="https://www.gstatic.com/devrel-devsite/prod/v0e0f589edd85502a40d78d7d0825db8ea5ef3b99ab4070381571c97ab7df9861/cloud/images/favicons/onecloud/favicon.ico">Google Cloud Vertex AI Enterprise</DocNode>, replacing the developer Gemini API. This provides automatic Application Default Credentials (ADC) token resolution, configurable safety thresholds above the default API limits, and enterprise-grade IAM-backed access control.

### Tink Envelope Cryptography & Post-Quantum Safeguards

Local patient records are encrypted at rest using **Google Tink AEAD** cryptographic envelopment. Transport security includes Quantum-Safe Cryptography (QSC) Kyber/Dilithium protocol fallbacks as a HIPAA-compliant future-proofing layer against post-quantum threats.

### Smiley Gesture Authentication & WebAuthn

A secure, intuitive access control screen replacing legacy numeric PIN entries. Clinicians draw a "smiley face" gesture on an interactive Canvas interface to unlock the workspace. To maintain multi-platform compatibility, the system features **device-supported biometric checks**, rendering a WebAuthn biometric unlock option only when the client device supports hardware-level biometric sensors.

### Shift-Left Pre-Commit Security Pipeline

A Husky pre-commit hook runs before every commit: TypeScript type checking, Vitest unit tests, markdown image reference scanning, and a credential/API key leak scanner that blocks commits containing secrets. This enforces security at the point of authorship rather than at deployment.

### IP-Based API Rate Limiting

A custom in-memory `rateLimiter` middleware on the Express backend mitigates denial-of-service and resource exhaustion on patient data and documentation endpoints, without requiring an external service.

### CodeQL-Hardened Routes

- **SSRF Prevention:** Strict model selection whitelist (`normalizeAndValidateModel`) blocks server-side request forgery in stream and chat paths.
- **Path Traversal Prevention:** `path.resolve` checks with strict parent-directory containment guard the static study docs router.
- **Clear-Text Logging Redaction:** PII and secret values are redacted from all console/CI log outputs in the HIPAA compliance scanner.
