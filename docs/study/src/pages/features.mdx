# Features

Every feature in Pocket Gull serves a clinical purpose. Nothing decorative, nothing superfluous.

## AI & Voice

### Live AI Consult & Multi-Agent Orchestration

Powered by <DocNode term="@google/adk" category="Google · Agent Development Kit" hint="Open-source TypeScript/Python framework for building multi-agent AI systems. Supports LlmAgent, SequentialAgent, and LoopAgent patterns with tool management." link="https://google.github.io/adk-docs/" linkLabel="ADK Docs →" icon="https://ai.google.dev/favicon.ico">`@google/adk`</DocNode> and the <DocNode term="Web Speech API" category="W3C · Web Platform" hint="W3C specification for speech recognition and synthesis in browsers. Chrome implementation by Google." link="https://developer.mozilla.org/en-US/docs/Web/API/Web_Speech_API" linkLabel="MDN Reference →" icon="https://developer.mozilla.org/favicon-48x48.png">Web Speech API</DocNode>. Specialized <DocNode term="LlmAgent" category="Google · Agent Development Kit" hint="Core agent class from @google/adk. Configured with a Gemini model, system prompt, and optional tools. Each focuses on a diagnostic lens." link="https://google.github.io/adk-docs/agents/" linkLabel="Agent Reference →" icon="https://ai.google.dev/favicon.ico">`LlmAgent`</DocNode> experts — each focused on a diagnostic lens (Overview, Interventions, Monitoring, Education) — synthesize clinical data into actionable insights through an interruptible, natural conversational UI.

The system maintains **context-aware memory** of recently discussed report nodes, enabling inline follow-up queries without re-stating context.

### Care Plan Recommendation Engine

A professional clinical analysis engine that synthesizes structured strategies for patient care. Output is organized by diagnostic lens, and each report node supports **inline agent queries** — click any recommendation to ask the AI directly about it.

### Orthomolecular Profiling & Biomarker Matrix

Automatically extracts and visualizes biochemical markers (e.g., Magnesium, B12, Homocysteine) from the AI-generated functional protocols. The glassmorphic dashboard uses pulsed indicators and semantic color-coding to provide an instant orthomolecular status read.

### Colleague Collaboration Room (TaskFlow)

A real-time multiplayer workspace integrated directly into the patient's view. Clinicians can share patient states, dictate clinical notes, and chat collaboratively without leaving the active diagnostic session.

### Hands-Free Voice Dictation & Controls

Speak the wake word **"Gull"** followed by actions (e.g., *"Gull, highlight head"*, *"Gull, sync data"*) during active dictation. The application intelligently intercepts these commands without modifying the transcript. Clinicians can also use the integrated dictation modal to add tasks or chat messages completely hands-free.

### Cognition & Multilingual Care Plans

Seamlessly translate Care Plans into dyslexia-friendly or pediatric formats, or professionally translate them into **Spanish, German, French, or Mandarin**. Outputted to PDF using <DocNode term="jsPDF" category="James Hall · Parallax" hint="Client-side PDF generation library for JavaScript. Generates printable clinical stationery and accessible export formats." link="https://github.com/parallax/jsPDF" linkLabel="GitHub →" icon="https://github.com/favicon.ico">jsPDF</DocNode> with refined Dieter Rams "carousel informatics" typography.

## Audio-Visual Stimulation (AVS)

### Multi-Layer Hemi-Sync Audio Engine

A high-fidelity 4-oscillator engine (via `GlobalAvsService`) supporting **phase-locked frequency entrainment** and a clinical-grade **Pink Noise synthesizer**. Carrier frequencies are tuned per clinical protocol; binaural beat offset is set by therapeutic target (e.g., Delta 2 Hz for sleep, Alpha 10 Hz for calm focus, Gamma 40 Hz for cognitive activation).

### Clinical AVS Breathing Protocols

The AVS Therapy panel generates a **Gemini-powered breathing protocol** matched to the patient's active conditions, wave state, and session intent. Protocols include breath rate (BPM), inhale/hold/exhale ratios, and clinical safety flags. An animated breath guide visualizes the cycle in real time.

### Athletic Enhancement AVS

Specialized protocols via `AthleticProtocolService` for performance and recovery contexts:

| Mode | Target | Key Frequencies |
|------|--------|-----------------|
| **Priming** | Pre-competition activation | Beta 18–22 Hz |
| **Flow State** | Deep focus / zone entry | Alpha 10 Hz |
| **Recovery** | Post-exertion restoration | Theta 6 Hz |
| **Phase-Shift** | Rapid state change | Gamma 40 Hz |

### DOC Stimulation Protocols

Structured auditory and vibroacoustic stimulation guidelines via `DocProtocolService` for **Disorders of Consciousness**: Coma, Vegetative State/Unresponsive Wakefulness Syndrome (VS/UWS), and Minimally Conscious State (MCS). Protocols reference evidence-based stimulation timing windows.

## Ambient Intelligence

### Circadian UI

A continuous, hardware-accelerated circadian CSS variable system (`CircadianSleepinessService`) smoothly transitions ambient UI colors based on the time of day — from cool dawn tones through peak daylight to warm evening amber — to offset clinician fatigue during extended shifts.

### KSS Readiness Gateway

The 9-point **Karolinska Sleepiness Scale (KSS)** is embedded in the secure splash screen, enabling a clinician alertness check before each session. The selected score overrides the circadian theme and can log a fatigue advisory to the session.

### IoT Smart Lighting Sync

`AmbientLightingService` mathematically maps the application's real-time circadian HSL values directly to local **Philips Hue** smart lights via the local network API (with a `hue-relay.js` proxy to resolve HTTPS mixed-content). The clinical environment physically co-regulates with the on-screen theme.

### Real-Time DSP Pipeline (FastAPI Sidecar)

A Python FastAPI sidecar (`pocketgull_api/`) processes real-time **HDF5 ECG streams** using SciPy to extract HRV (RMSSD) and respiratory frequency metrics, feeding them back into the circadian and AVS systems.

## Data & Export

### FHIR-Standard Data Portability

Real-time persistence with visual "Saving…" / "Saved ✔" indicators. Patient data is exported via <DocNode term="FHIR R4 Bundles" category="HL7 International" hint="Fast Healthcare Interoperability Resources Release 4 — the international standard for electronic health data exchange." link="https://hl7.org/fhir/R4/bundle.html" linkLabel="HL7 FHIR R4 →" icon="https://hl7.org/favicon.ico">**Unicode-safe Base64 encoded FHIR Bundles**</DocNode>, ensuring interoperability with compliant healthcare systems.

### FHIR Import Service

Ingest external healthcare payloads — including **Epic/MyChart "Lucy" format** — mapping `DiagnosticReport`, `Observation`, and `Condition` FHIR resources into Pocket Gull's native data models.

### Patient Management System

Full CRUD capabilities for patient records, including historical visit review, permanent record removal, and multi-patient switching via a dropdown selector.

### Local-First PWA Architecture

Pocket Gull operates natively as a Progressive Web App (PWA). Under poor or offline connectivity, the system degrades gracefully — persisting data via `localStorage` while providing clear status indicators for temporarily unavailable AI cloud features.

## UX & Accessibility

### Minimalist Dieter Rams Design

A premium, minimalist UI prioritizing **clarity, neutrality, and functional excellence**. Seamless mobile responsive layouts using `100dvh`. Dark-mode agent conversations provide comfortable extended-use scenarios.

### Interactive Tour & Help

A 9-step interactive walkthrough launches automatically on first visit and can be replayed at any time via the **?** help button. Each step spotlights a UI element with an SVG cutout, a floating tooltip card, and a progress indicator.

### Smartwatch & Mobile Optimization

Responsive Two-Column Grid UI scaling down to extremely constrained viewports — as small as **286px width** (Pixel Watch 2) — for ultra-portable clinical referencing.

### Box Breathing UX

Focused 16-second box breathing visual animations integrated into primary intake text areas. Promotes **practitioner mindfulness** during high-stress clinical encounters.

### Interactive Task Bracketing

Rapidly mark up generated care plans using a **double-click state machine**: Normal → Added → Removed. The core human-in-the-loop control mechanism — no AI recommendation proceeds without clinician vetting.

### Offline PWA & Gemini Nano

The `HybridProvider` routes chat and conversational queries to Chrome's on-device `window.ai` (Gemini Nano) when offline or for token-free local processing.
