## Data Flow

1. **Input** вҖ” Clinician enters data via body map interaction, intake forms, or voice dictation (<DocNode term="Web Speech API" category="W3C В· Web Platform" hint="W3C standard browser API for speech recognition and synthesis. No external dependencies. Supported in Chrome, Edge, and Safari." link="https://developer.mozilla.org/en-US/docs/Web/API/Web_Speech_API" linkLabel="MDN Reference вҶ’" icon="https://developer.mozilla.org/favicon-48x48.png">Web Speech API</DocNode>).
2. **State** вҖ” All inputs flow into the centralized `PatientState` service, which uses <DocNode term="Angular Signals" category="Google В· Angular Team" hint="Fine-grained reactive primitives. signal() creates writable state, computed() derives values, effect() triggers side-effects. Powers all UI reactivity in Pocket Gull." link="https://angular.dev/guide/signals" linkLabel="angular.dev вҶ’" icon="https://angular.dev/favicon.ico">Angular Signals</DocNode> for granular reactivity.
3. **Analysis** вҖ” The Analysis component reads state and invokes the <DocNode term="ADK InMemoryRunner" category="Google В· Agent Development Kit" hint="Google's open-source framework for building multi-agent AI systems. InMemoryRunner manages agent lifecycle, tool calls, and LLM orchestration in-process." link="https://google.github.io/adk-docs/" linkLabel="ADK Docs вҶ’" icon="https://ai.google.dev/favicon.ico">ADK `InMemoryRunner`</DocNode>, which orchestrates specialized <DocNode term="LlmAgent" category="Google В· Agent Development Kit" hint="Core agent class from @google/adk. Configured with a model, system prompt, and optional tools. Each instance focuses on one diagnostic lens." link="https://google.github.io/adk-docs/agents/" linkLabel="Agent Reference вҶ’" icon="https://ai.google.dev/favicon.ico">`LlmAgent`</DocNode> experts.
4. **Generation** вҖ” Each agent queries <DocNode term="Gemini 2.5 Flash" category="Google DeepMind" hint="Google's fast, cost-efficient multimodal model. Optimized for real-time streaming with low latency. Supports text, images, audio, and video inputs." link="https://ai.google.dev/gemini-api/docs/models#gemini-2.5-flash" linkLabel="Model Reference вҶ’" icon="https://ai.google.dev/favicon.ico">`gemini-2.5-flash`</DocNode>, which streams structured JSON chunks back through the runner.
5. **Output** вҖ” Streamed chunks are rendered in real-time as a Care Plan organized by diagnostic lens.
6. **Persistence** вҖ” Patient state is saved to local session storage with visual "SavingвҖҰ/Saved вң”" indicators.
7. **Export** вҖ” Data is exportable as <DocNode term="FHIR R4" category="HL7 International" hint="Fast Healthcare Interoperability Resources Release 4 вҖ” the current standard for electronic health data exchange. Defines resource types like Patient, Condition, and Observation." link="https://hl7.org/fhir/R4/" linkLabel="HL7 FHIR R4 вҶ’" icon="https://hl7.org/favicon.ico">FHIR R4 Bundles</DocNode> (Base64 JSON) or printable PDF stationery.

## Service Orchestration & Multi-Agent Parallelism

Pocket Gull coordinates complex clinical synthesis through a structured, multi-agent orchestration layer designed for low-latency feedback and clinical modularity:

### 1. Parallel Lens Generation
Rather than relying on a single, monolithic LLM request, the `ClinicalIntelligenceService` partitions the Care Plan into six independent clinical dimensions (lenses). These lenses are processed concurrently in the background using `Promise.allSettled()` orchestration:
* **Summary Overview** вҖ” Unified clinical assessment, urgency priority list, and measurable timeline goals.
* **Functional Protocols** вҖ” Biochemical pathway targets, precision molecule dosing matrices, and HPA axis/circadian guidelines.
* **Nutrition** вҖ” Micronutrient deficiency correction, cellular oxidative stress, and whole-food adjustments.
* **Orthomolecular Profiling** вҖ” Methylation blocks, heavy metal depletions, and structured JSON biomarker output mapping.
* **Monitoring & Follow-up** вҖ” Time-horizon next steps, tracking parameters, and threshold escalation triggers.
* **Patient Education** вҖ” Empathetic, de-identified plain-language translations of all recommended strategies.

### 2. Avian Agent Personas
Each clinical lens is orchestrated by a dedicated specialist agent from the *Gull Squadron* with customized prompt boundaries:
* `Gulliver` (Summary Overview) вҖ” Synthesis and big-picture overview lead.
* `Swoop` (Functional Protocols, Nutrition, Orthomolecular Profiling) вҖ” Precision pathway and dosing specialist.
* `Sentinel` (Monitoring & Follow-up) вҖ” Care coordination and vigilance monitor.
* `Scribes` (Patient Education) вҖ” Patient-facing plain language translation specialist.

### 3. Dynamic Prompt Compounding
The system compiles system instructions dynamically at runtime by stitching together several layers:
1. **Agent Identity** вҖ” Persona role, voice, and diagnostic boundaries.
2. **Clinical Paradigm** вҖ” Standard Western guidelines, Eastern (Zang-Fu, 8 Principles, acupoints, pulse), or Ayurvedic (Dosha constitutions, Agni/Ama metabolism, Dhatu tissue mapping) paradigms.
3. **ORCID Research Integration** вҖ” Dynamically appends the clinician's publications and research keywords to align AI insights with the practitioner's scholarly background.
4. **Formatting Constraints** вҖ” Strict Markdown layouts, citation parameters, and HIPAA de-identification masks.

### 4. Real-time Reactive Streaming
Each parallel agent execution maps directly to a chunked generator stream (`generateReportStream$`). Chunks are consumed as they arrive from Vertex AI and immediately update the respective Angular Signals. This drives fine-grained UI updates card-by-card in real-time, keeping the interface responsive and alive.
