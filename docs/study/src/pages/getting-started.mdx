## Installation

```bash
# Clone the repository
git clone https://github.com/philgear/pocket-gull.git
cd pocket-gull

# Install dependencies
npm install
```

---

## Development

```bash
# Start the development server (client + SSR Express)
npm run dev
```

The application will be available at `http://localhost:4200`. Enter your Gemini API key on the splash screen, or press **Try Demo** to load a pre-sampled patient with example AI analysis outputs.

> **First-run tour:** On your first visit, Pocket Gull launches a 9-step interactive walkthrough. You can replay it anytime from the **?** help button in the header.

## Environment Variables

Create a `.env.local` file in the project root (or in `pocketgull_api/` — the server searches both):

```bash
GEMINI_API_KEY=your_key_here

# Optional — Philips Hue ambient sync
HUE_BRIDGE_IP=192.168.x.x
HUE_API_KEY=your_hue_key

# Optional — Google Programmable Search (evidence panel)
GOOGLE_CSE_ID=your_cse_id
GOOGLE_CSE_KEY=your_cse_key
```

## Production Build

```bash
# Build for production (Angular SSR + esbuild)
npm run build

# Preview the production build locally
npm run preview
```

The production build outputs to `dist/app/` with full <DocNode term="Angular SSR" category="Google · Angular Team" hint="Server-Side Rendering — Angular renders the initial page on the Express.js server, then hydrates client-side. Faster first paint and SEO." link="https://angular.dev/guide/ssr" linkLabel="SSR Guide →" icon="https://angular.dev/favicon.ico">SSR</DocNode> support. An `@mlc-ai/web-llm` browser shim at `src/shims/url-shim.ts` redirects the Node.js `url` module for browser-safe bundling.

### Live Deployment

- **Primary URL:** [pocketgull.app](https://pocketgull.app)
- **Alt URL:** [pocketgall.app](https://pocketgall.app)

### Keyless CI/CD (Workload Identity Federation)

The repository uses **Workload Identity Federation** for keyless, HIPAA-aligned CI/CD — no long-lived service account keys are ever stored in GitHub Secrets.

```yaml
# .github/workflows/deploy.yml (excerpt)
- uses: google-github-actions/auth@v2
  with:
    workload_identity_provider: projects/$PROJECT_NUMBER/locations/global/workloadIdentityPools/github-pool/providers/github-provider
    service_account: github-deployer@$PROJECT_ID.iam.gserviceaccount.com
```

### One-Click Manual Deploy

```bash
npm run deploy
# Equivalent to: gcloud run deploy pocket-gull --source . --region us-central1
```

## Security Hardening

Pocket Gull implements defense-in-depth security for a clinical platform:

- **HTTPS Enforcement:** HSTS with `max-age=31536000; includeSubDomains` served by Express
- **Content Security Policy:** Strict CSP with nonce-based inline script allowance
- **CORS Allowlist:** Restricted to `pocketgull.app`, `pocketgall.app`, and `localhost`
- **Automated Red-Teaming:** Vitest suite in `tests/safety.spec.ts` verifies Gemini `BLOCK_MEDIUM_AND_ABOVE` safety thresholds
- **Zero PII Persistence:** All patient state is transient or stored only in `localStorage` — no server-side PHI

See [`SECURITY.md`](../../SECURITY.md) for the full security policy and vulnerability disclosure process.
